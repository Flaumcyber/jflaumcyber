# Penetration Testing Report — Attack from End User's Desktop

**Network Security Final Project — Oklahoma State University Institute of Technology, Fall 2023**

As the team lead, I oversaw the planning, research, and development of a simulated penetration test targeting a large retail organization's network (modeled after Costco). The objective was to evaluate how an attacker could compromise an enterprise network by exploiting human behavior, endpoint security weaknesses, and web application vulnerabilities.

## Project Objectives

- Assess the security risks associated with end-user workstations.
- Simulate realistic attack paths that could lead to unauthorized access.
- Evaluate existing security controls and recommend improvements.

## Attack Scenarios Analyzed

The team researched and documented several attack vectors, including:

- **USB-based attacks** — Simulated how a malicious USB device could be used to compromise an employee workstation after physical access.
- **Social engineering and spear phishing** — Examined how targeted phishing campaigns could obtain employee credentials or deliver malware.
- **Session hijacking concepts** — Discussed how attackers could abuse compromised authentication sessions after successful phishing.
- **SQL injection** — Evaluated how poorly validated web application inputs could expose backend databases and sensitive customer information.
- **Privilege escalation and lateral movement** — Demonstrated how an initial compromise could be expanded to access additional systems and sensitive organizational data.

## Key Findings

The report concluded that organizations remain vulnerable when technical controls are combined with human error. Major risks identified included:

- Weak input validation leading to web application attacks.
- Susceptibility to phishing and social engineering.
- Inadequate endpoint protection against removable media.
- Insufficient employee cybersecurity awareness.
- Opportunities for attackers to move laterally after an initial compromise.

## Security Recommendations

The team recommended several defensive improvements:

- Implement rigorous input validation to prevent SQL injection.
- Strengthen employee security awareness and anti-phishing training.
- Enhance email security and enforce multi-factor authentication.
- Conduct regular vulnerability assessments and timely patch management.
- Foster an organization-wide culture of cybersecurity awareness.

This project reinforced that cybersecurity is about much more than deploying security tools. Even organizations with technologies like multi-factor authentication and endpoint protection can still be vulnerable if employees are not properly trained or if security best practices are overlooked. It taught me the importance of taking a layered defense approach that combines secure system design, continuous monitoring, employee awareness, regular patch management, and proactive security assessments. As the team leader, I also learned the value of effective communication, delegation, and coordinating multiple team members to deliver a comprehensive technical report.
